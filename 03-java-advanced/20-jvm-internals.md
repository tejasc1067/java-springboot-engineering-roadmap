# JVM Internals in Java

Understanding JVM internals is extremely important for backend engineering.

Modern backend systems heavily depend on:
- JVM performance
- runtime optimization
- memory behavior
- JIT compilation
- scalable execution

JVM understanding transforms developers into JVM-aware backend engineers.

---

# 1. What is JVM?

JVM:
# Java Virtual Machine

Responsible for:
- executing bytecode
- memory management
- garbage collection
- runtime optimization

Very important Java foundation.

---

# 2. Why JVM Exists?

JVM enables:
# platform independence

Java compiles into:
# bytecode

which JVM executes.

Very important runtime architecture concept.

---

# 3. Class Loading Process

Before execution:
# classes must load into memory

Very important runtime lifecycle topic.

---

# 4. Class Loaders

JVM uses:
- Bootstrap ClassLoader
- Platform ClassLoader
- Application ClassLoader

Very important JVM architecture concept.

---

# 5. Class Loading Phases

Major phases:
- Loading
- Linking
- Initialization

Very important interview topic.

---

# 6. Bytecode

Java compiles into:
# bytecode

Stored inside:
# .class files

Very important JVM execution concept.

---

# 7. Execution Engine

JVM execution engine:
# runs bytecode

Uses:
- interpreter
- JIT compiler

Very important runtime optimization topic.

---

# 8. Interpreter vs JIT

Interpreter:
- executes bytecode line-by-line

JIT:
- compiles hot code into native machine code

Very important backend optimization concept.

---

# 9. HotSpot JVM

JVM detects:
# frequently executed code

called:
# hot methods

Very important runtime optimization topic.

---

# 10. Stack Frames

Each method call creates:
# stack frame

Contains:
- local variables
- operand stack
- method metadata

Very important runtime execution concept.

---

# 11. Method Invocation

Method execution internally involves:
- frame creation
- stack manipulation
- bytecode execution

Very important JVM internals topic.

# 12. JVM Runtime Areas

Important runtime areas:
- Heap

- Native Method Stack

Very important JVM architecture foundation.
