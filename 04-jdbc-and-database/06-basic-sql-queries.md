# Basic SQL Queries

This topic focuses on:
# practical SQL querying foundations

Backend systems constantly perform:
# data retrieval and manipulation

Very important backend engineering topic.

---

# SELECT Query

Used for:
# retrieving data

Example:

SELECT * FROM users;

Most important SQL operation.

Very important backend persistence concept.

---

# Selecting Specific Columns

Instead of retrieving everything:
# retrieve only required data

Example:

SELECT name, email FROM users;

Very important performance-awareness mindset.

---

# Why SELECT Matters

Backend systems constantly fetch:
- users
- orders
- products
- payments
- inventory

Very important backend engineering workflow.

---

# INSERT Query

Used for:
# storing data

Example:

INSERT INTO users(name, email)
VALUES('Tejas', 'tejas@email.com');

Very important persistence workflow.

---

# UPDATE Query

Used for:
# modifying existing data

Example:

UPDATE users
SET name = 'Updated Name'
WHERE id = 1;

Very important transactional-system concept.

---

# DELETE Query

Used for:
# removing data

Example:

DELETE FROM users
WHERE id = 1;

Very important backend persistence operation.

---

# Why WHERE Clause Matters

Without filtering:
- unnecessary data retrieval
- performance problems
- dangerous updates/deletes

Very important production-awareness concept.

---

# CRUD Workflow in Backend Systems

Backend APIs fundamentally depend on:
